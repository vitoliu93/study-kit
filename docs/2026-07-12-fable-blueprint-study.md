## 图纸 1 · concept-note 桥 — 存档里讲清过的概念,誊一份进知识库

服务信件种子二。触发事件(信里定的):**哪天你想引用自己的 micrograd 洞见却翻不到。**

### 草稿(resume-learning SKILL.md,模式一第 4 步后拟插入 4.5)

```markdown
4.5 **概念誊录(静默,零提问)。** 本次存档里若有"讲清了"的概念——判定来源:
    第 2 步主动回忆追问里答上来的项,减去第 3 步 L2 体检标了假懂/漂移的项
    (L2 只产负向 flag,正向信号只能来自追问答复;两步都不重新问)——AI 顺手把
    用户的原话解释誊成一小段笔记,写入 qmd 索引根下 `learning/<概念-slug>.md`
    (根 = `qmd collection show` 的实际目录,首次使用时钉死写进本节):
    首行 = 用户自己的比喻/解释(保留原话腔,不转述成教科书),次行 = 出处
    (日期 + 主线 + RESUME 历史行)。规则:一次最多一条;拿不准"是否真讲清"就不誊;
    誊了只在收尾确认里多报一个文件名,不展示内容、不追问。RESUME.md 仍是唯一存档,
    笔记是导出物——丢了也不影响读档。
```

### 设计决策

- **判定复用第 3 步的 L2 体检结果**,不新增提问——这是过北极星裁决的关键:存档流程的
  用户侧动作数为零增。
- **保留原话腔**:你的"信贷员/审批经理"比喻比任何教科书定义对未来的你更有召回价值。
  费曼判据的产物就该以费曼的形式存。
- **"笔记是导出物"一句是安全阀**:RESUME.md 的唯一性(你们家规则第 1 条)不被动摇,
  桥断了学习系统无感。
- 与 dev-kit 蓝图图纸 2(`note` skill)同构不同源:那边人工触发记工作结论,落
  `notes/<YYYY-MM>/`;这边搭存档顺风车记学习概念,落 `learning/<slug>`。同一 qmd 根下的
  **并列子树**,分工不是竞争——采纳任何一边时,先跑一次 `qmd collection show` 钉死根路径,
  誊完第一条立刻 `qmd query` 验证真的能召回,再算装好。

### 采纳条件

- 触发事件发生的那天。装之前先手动誊一条(就誊"打分卡比喻")体验一次全流程,
  确认"零提问"在实操里成立,再动 SKILL.md。

## 图纸 2 · v2 立项书 — tiny-distill(学习目标行的下一站,不是插件功能)

服务信件种子一。这张图**不属于 study-kit 插件**——按"插件装机制,项目装目标"的分层,
它属于 self-llm/RESUME.md 的目标行。画在这里只因为它是学习域的图纸。
触发事件:**v1 收官那天**(从零训出小 GPT 且能讲清每一步)。

### 草稿(拟写入 self-llm/RESUME.md 目标行的 v2 展开,以及一页立项书)

```markdown
# tiny-distill — 用自己训的本地小模型接管 ccobs 会话蒸馏

## 为什么是它(动机对齐目标行)
v2 判据要求"用本地小模型解决一个自己的真实问题"。ccobs 的蒸馏 provider 链
(deepseek > gemini > openrouter > lmstudio)最后一级本来就是本地插座:
接管它 = 学习系统的出口焊进工作系统的入口。

## 数据(零标注,但有一步构造)
输出侧现成:obs.db 的 observations 表存着 1300+ 条蒸馏结果(summary/task_type/outcome),
且每天在长。输入侧(会话原文)不在这张表里——要按 session_id 回原始 JSONL 重建、
切片、对齐成训练对。零标注不等于零工序:数据构造本身就是阶段 1 的第一个产出物
(顺便逼你真的理解"训练对长什么样")。

## 阶段(每段都有产出物 + 能复述的判据,15 分钟粒度拆步交给 coach)
1. 基线:挑 20 个 held-out 会话,lmstudio 现成小模型 zero-shot 蒸馏,
   和账本里 deepseek 的产出盲评对比——先知道"不微调差多远"。
2. 微调:MLX LoRA(qmd 已收 llm-finetuning-primer,含"哪些要重学"表),
   训练对来自阶段 0 导出。产出物 = 一个能跑的本地蒸馏模型。
3. 上岗:provider 链头部换成它跑一周,质量不塌、成本为零。
4. 复述判据:能讲清数据构造/LoRA 在哪一层动了什么/为什么小模型够用——
   讲不清就不算学会,回炉。

## 边界
不做通用助手、不做多任务——只蒸馏这一件事。模型小、任务窄、数据自产,
这是"第一个真实问题"的最短路径。
```

### 设计决策

- **每个阶段自带"产出物 + 能复述"双判据**,直接继承目标行契约的格式——立项书本身
  就是按你的北极星语法写的。
- 阶段 1 先测 zero-shot 基线,是给"也许根本不用微调"留门:如果基线够用,v2 提前完成,
  学习收益不减(你仍要能讲清为什么够用)。诚实的立项书要包含"目标可能提前作废"的路径。
- 数据自产(obs.db)避开了业余项目最常见的死法:卡在找数据集。

### 采纳条件

- v1 收官时,把本图纸的一句话版写进目标行 v2 位;立项书全文到时候重写——
  那时的你比今天的我更清楚哪里该改。

## 图纸 3 · 取证降级契约(resume-learning 三行补丁)

服务信件第二节第二声低语。触发事件:**harvest_sessions.py 第一次真的坏**
(某家 CLI 改了会话格式/路径)。

### 草稿(resume-learning SKILL.md,**两个调用点都要**:模式一第 1 步取证、模式二第 2 步)

```markdown
取证脚本报错或超 10 秒未返回 → 一句话说明("取证脚本失效,本次按实际文件/RESUME.md
继续,[补登] 靠口头"),照常走后续步骤。存档与读档都永不因取证阻塞;
脚本修不修、何时修,是学习会话之外的事。
```

### 设计决策

- 10 秒硬顶 + 一句话降级,把"活动部件坏掉"的代价钉死在一句话——30 秒承诺的
  防波堤。修脚本被明确划出学习会话("工具建设是最高级的拖延"的执行细则)。
- 初稿只补了模式二,审图时被抓:模式一第 1 步的取证是**无条件**调用点(次次跑),
  模式二反而是间隔 >1 天才触发——降级契约漏掉高频路径等于没写。两处共用同一句话。

### 采纳条件

- 它第一次坏的那天。如果实测发现现有行为已经等价降级,把这张图纸撕掉并在
  commit message 里记"验证过,天然免疫"。

## 图纸 4 · 存档历史归档契约(resume-learning 补丁)

审图员补画的一张,我核过后收进来:信第二节第三声低语(打鱼记录 vs 30 秒承诺)是
唯一被信指认、却没被画出图纸的未来补丁——而它保护的正是北极星度量本身。

### 草稿(resume-learning SKILL.md,模式二第 1 步 + 规则末条补)

```markdown
读档只读 RESUME.md 顶部三节(我在哪/下一步/卡点)+ 存档历史最近 N 行;更早的行
整段移到同目录 `RESUME.archive.md`(照旧只追加、只增不删),RESUME.md 底部留一行
「更早历史见 RESUME.archive.md」。读档永不遍历全历史。
```

### 设计决策

- **归档 ≠ 删。** 打鱼记录的情绪价值建立在完整上——完整性整体移进 archive 保留,
  "只增不删"守住,只把 30 秒读档从行数增长里解耦。
- archive 留在同一学习项目目录,取证/召回不丢。
- **触发是体感不是数字**:读档实测超 30 秒、或历史把顶部三节挤出一屏的那天——
  不设行数阈值、不设日期节律(你的红队原话:数字节律是死条款)。

### 采纳条件

- 读档第一次实测超过 30 秒那天。在此之前存档历史照常长,一行不动;若届时发现
  "读档只读顶部三节"已天然免疫,撕掉本图并在 commit 里记"验证过,免疫"。

## 废稿 · 一切"让会话发生"的技术解

信里说过,这里正式画废留档:提醒推送、打卡、streak 计数、日历钩子、
"今天该学习了"的任何变体。

理由只有一条但够硬:红队已把"激活缺口"划为结构性够不到——插件在会话开始后才存在。
所有此类设计都在假装够得到,实际效果是把工具变成愧疚制造机,直接违反北极星
("凡增加下次启动成本——**操作上的或情绪上的**——一律否决")。情绪成本那半句
是这条北极星最锋利的部分,别让任何聪明设计钝化它。

学习系统里唯一合法的"拉力"是图纸 2 那种:让目的地真实可见。
拉力来自想去的地方,不来自闹钟。
